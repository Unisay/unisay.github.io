Interests

Pure FP as production paradigm

Pure functional programming is not a research curiosity — it is a production discipline. Haskell and PureScript are my primary tools precisely because they make a class of bugs impossible by construction, not because they are fashionable.

Bridging research and engineering

I author CIPs, build benchmarking frameworks, write simulations, and wire LLMs into static analysis pipelines. The distance between "how does this work" and "this is in production" should be as short as possible.

Teaching and durable infrastructure

Three colleagues learned Haskell in production teams because I taught them. The serverless services I built in 2019 are still running without incidents. I measure success by what survives, not what ships.
